# **第一章 数据库概述**

## **1.1 数据库基本信息**

> 数据库名称：inventory_v2
>
> 数据库版本：MySQL 8.0.33
>
> 字符集：utf8mb4
>
> 排序规则：utf8mb4_unicode_ci

## **1.2 设计原则**

> 主键统一命名：id，BIGINT类型，自增
>
> 通用字段：create_time, update_time, create_by, update_by, is_deleted
>
> 逻辑删除：is_deleted = 1 表示已删除
>
> 外键约束：使用逻辑外键，不使用物理外键

## **1.3 数据库ER图说明**

本系统数据库按照业务模块划分为系统权限模块、基础档案模块和核心业务模块三大模块。各模块之间通过逻辑外键关联，形成完整的数据关系网络。

> 系统权限模块包含用户、部门、角色、菜单等基础权限数据，为整个系统提供权限控制支撑。
>
> 基础档案模块包含商品、仓库、供应商、客户等业务基础数据，为业务流转提供数据基础。
>
> 核心业务模块包含库存管理、采购管理、销售管理等业务数据，实现企业的核心业务流程。

## **7.2 常用SQL示例**

以下为系统中常用的SQL查询示例：

> 1\. 查询用户列表
>
> SELECT id, username, real_name, phone, email, dept_id, status FROM
> sys_user WHERE is_deleted = 0 ORDER BY create_time DESC;
>
> 2\. 查询菜单树
>
> SELECT id, parent_id, menu_name, menu_code, menu_type, path, icon,
> order_num FROM sys_menu WHERE is_deleted = 0 AND status = 0 ORDER BY
> order_num;
>
> 3\. 查询库存信息
>
> SELECT sm.id, w.warehouse_name, ps.sku_name, sm.quantity,
> sm.available_qty FROM stock_main sm LEFT JOIN base_warehouse w ON
> sm.warehouse_id = w.id LEFT JOIN base_product_sku ps ON sm.sku_id =
> ps.id WHERE sm.is_deleted = 0;
>
> 4\. 查询采购订单
>
> SELECT po.id, po.order_no, s.supplier_name, po.total_amount,
> po.order_status, po.order_date FROM bus_purchase_order po LEFT JOIN
> base_supplier s ON po.supplier_id = s.id WHERE po.is_deleted = 0 ORDER
> BY po.create_time DESC;
>
> 5\. 查询销售订单
>
> SELECT so.id, so.order_no, c.customer_name, so.total_amount,
> so.order_status, so.order_date FROM bus_sale_order so LEFT JOIN
> base_customer c ON so.customer_id = c.id WHERE so.is_deleted = 0 ORDER
> BY so.create_time DESC;
>
> 6\. 查询库存预警
>
> SELECT sm.id, ps.sku_name, w.warehouse_name, sm.quantity,
> sm.warning_qty FROM stock_main sm LEFT JOIN base_product_sku ps ON
> sm.sku_id = ps.id LEFT JOIN base_warehouse w ON sm.warehouse_id = w.id
> WHERE sm.quantity \<= sm.warning_qty AND sm.is_deleted = 0;
