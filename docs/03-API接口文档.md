# 第一章 接口规范说明

## 1.1 接口基础信息

  ----------------------------------- -----------------------------------
  **基础路径**                        http://localhost:8080

  **协议**                            HTTP/HTTPS

  **数据格式**                        JSON

  **字符编码**                        UTF-8

  **接口风格**                        RESTful

  **认证方式**                        JWT Bearer Token
  ----------------------------------- -----------------------------------

## 1.2 认证方式

本系统采用JWT（JSON Web Token）进行身份认证。

**认证类型：**JWT Bearer Token

**Token获取：**通过登录接口返回

**Token使用：**请求头 Authorization: Bearer {token}

## 1.3 统一响应格式

所有接口返回统一的JSON格式数据：

+-----------------------------------------------------------------------+
| {                                                                     |
|                                                                       |
| \"code\": 200,                                                        |
|                                                                       |
| \"msg\": \"操作成功\",                                                |
|                                                                       |
| \"data\": {}                                                          |
|                                                                       |
| }                                                                     |
+-----------------------------------------------------------------------+

## 1.4 分页响应格式

列表查询接口返回分页数据格式：

+-----------------------------------------------------------------------+
| {                                                                     |
|                                                                       |
| \"code\": 200,                                                        |
|                                                                       |
| \"msg\": \"操作成功\",                                                |
|                                                                       |
| \"rows\": \[\],                                                       |
|                                                                       |
| \"total\": 100                                                        |
|                                                                       |
| }                                                                     |
+-----------------------------------------------------------------------+

## 1.5 错误码说明

  -----------------------------------------------------------------------
  **错误码**                          **说明**
  ----------------------------------- -----------------------------------
  200                                 操作成功

  400                                 参数错误

  401                                 未授权/Token失效

  403                                 无权限

  500                                 服务器内部错误
  -----------------------------------------------------------------------

## 9.2 在线调试说明

1\. 打开 Swagger UI 页面，查看所有可用接口。

2\. 点击接口展开详情，查看请求参数和响应格式。

3\. 点击 \"Authorize\" 按钮，输入 Bearer Token 进行认证。

4\. 点击 \"Try it out\" 按钮，填写参数后执行请求。

5\. 查看响应结果，确认接口调用是否成功。

**注意事项：**

• 需要先通过登录接口获取 Token，才能调试需要认证的接口。

• Token 有效期默认为 2 小时，过期后需重新登录获取。

• 部分 DELETE 接口支持批量操作，多个 ID 用逗号分隔。
