**第一章 部署前准备**

**1.1 系统要求**

在开始部署之前，请确保您的计算机满足以下系统要求：

> • 操作系统：Windows 10/11 (64位)
>
> • 内存：最低4GB，推荐8GB以上
>
> • 磁盘空间：至少10GB可用空间
>
> • 处理器：Intel Core i3 或同等性能以上
>
> • 网络：局域网环境（如需多用户访问）

**1.2 需要下载的软件清单**

请提前下载以下软件安装包：

  ------------------------------------------------------------------------
  **软件**          **版本**          **用途**          **下载地址**
  ----------------- ----------------- ----------------- ------------------
  JDK               1.8.0_301         Java运行环境      oracle.com

  MySQL             8.0.33            数据库            mysql.com

  Node.js           16.20.2           前端运行环境      nodejs.org

  Maven             3.6.3             后端构建工具      maven.apache.org

  Redis替代方案     Caffeine(内置)    本地缓存          无需安装
  ------------------------------------------------------------------------

注意：Redis替代方案使用内置的Caffeine本地缓存，无需额外安装Redis服务，大大简化了部署流程。

**1.3 软件安装步骤**

**1.3.1 JDK安装与环境变量配置**

步骤1：双击运行 jdk-8u301-windows-x64.exe 安装程序

步骤2：选择安装目录（建议使用默认路径）

步骤3：配置环境变量

> • 新建系统变量 JAVA_HOME，值为JDK安装路径（如：C:\\Program
> Files\\Java\\jdk1.8.0_301）
>
> • 编辑系统变量 Path，添加 %JAVA_HOME%\\bin

步骤4：验证安装 - 打开命令提示符，输入 java -version

**1.3.2 MySQL安装与配置**

步骤1：运行 mysql-installer-community-8.0.33.msi

步骤2：选择 \"Developer Default\" 或 \"Server only\" 安装类型

步骤3：设置root用户密码（请牢记此密码，后续配置需要使用）

步骤4：配置MySQL服务为Windows服务，并设置为自动启动

步骤5：验证安装 - 打开命令提示符，输入 mysql -u root -p

**1.3.3 Node.js安装**

步骤1：运行 node-v16.20.2-x64.msi 安装程序

步骤2：选择安装目录（建议使用默认路径）

步骤3：安装完成后，npm会自动安装

步骤4：验证安装 - 打开命令提示符，输入 node -v 和 npm -v

**1.3.4 Maven安装与配置**

步骤1：解压 apache-maven-3.6.3-bin.zip 到指定目录（如：D:\\maven）

步骤2：配置环境变量

> • 新建系统变量 MAVEN_HOME，值为Maven解压路径
>
> • 编辑系统变量 Path，添加 %MAVEN_HOME%\\bin

步骤3：验证安装 - 打开命令提示符，输入 mvn -v

**第二章 数据库初始化**

**2.1 创建数据库**

**2.1.1 启动MySQL服务**

方式一：通过服务管理器

> • 按 Win + R，输入 services.msc
>
> • 找到 MySQL 服务，确保状态为\"正在运行\"

方式二：通过命令行

> • 以管理员身份运行命令提示符
>
> • 输入：net start mysql

**2.1.2 使用Navicat或命令行创建数据库**

方式一：使用Navicat图形化工具

> • 打开Navicat，连接到MySQL服务器
>
> • 右键点击连接名，选择\"新建数据库\"
>
> • 数据库名：inventory_v2
>
> • 字符集：utf8mb4
>
> • 排序规则：utf8mb4_unicode_ci

方式二：使用命令行

> • 打开命令提示符，输入：mysql -u root -p
>
> • 输入密码后，执行以下SQL命令：
>
> CREATE DATABASE inventory_v2 CHARACTER SET utf8mb4 COLLATE
> utf8mb4_unicode_ci;

**2.2 导入初始化脚本**

步骤1：定位SQL脚本文件

> 脚本位置：inventory-backend/sql/inventory_v2.sql

步骤2：使用Navicat导入

> • 右键点击 inventory_v2 数据库
>
> • 选择\"运行SQL文件\"
>
> • 选择 inventory_v2.sql 文件
>
> • 点击\"开始\"执行导入

步骤3：或使用命令行导入

> mysql -u root -p inventory_v2 \< inventory_v2.sql

**2.3 验证数据库**

步骤1：检查表结构

导入成功后，数据库应包含以下表（共20张表）：

sys_user（用户表）、sys_role（角色表）、sys_menu（菜单表）、sys_dept（部门表）、sys_user_role（用户角色关联表）、sys_role_menu（角色菜单关联表）、sys_oper_log（操作日志表）、base_category（分类表）、base_product_spu（产品SPU表）、base_product_sku（产品SKU表）、base_warehouse（仓库表）、base_supplier（供应商表）、base_customer（客户表）、bus_purchase_order（采购订单表）、bus_purchase_item（采购明细表）、bus_sale_order（销售订单表）、bus_sale_item（销售明细表）、bus_stock_check（盘点单表）、bus_stock_check_item（盘点明细表）、stock_record（库存记录表）

**第三章 后端部署**

**3.1 配置数据库连接**

步骤1：打开配置文件

> 文件位置：inventory-backend/src/main/resources/application-druid.yml

步骤2：修改数据库连接配置

> spring: datasource: druid: driver-class-name: com.mysql.cj.jdbc.Driver
> url:
> jdbc:mysql://localhost:3306/inventory_v2?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=false&serverTimezone=Asia/Shanghai
> username: root password: 您设置的MySQL密码

配置项说明：

> • url：数据库连接地址，localhost表示本机，3306为MySQL默认端口
>
> • username：数据库用户名，默认为root
>
> • password：数据库密码，请修改为您设置的密码

**3.2 编译打包**

步骤1：打开命令提示符

> 切换到后端项目目录：cd inventory-backend

步骤2：执行Maven打包命令

> mvn clean package -DskipTests

参数说明：

> • clean：清理之前的构建文件
>
> • package：打包项目
>
> • -DskipTests：跳过测试（加快打包速度）

步骤3：等待打包完成

> • 首次打包会下载依赖，可能需要较长时间
>
> • 打包成功后，在 target 目录下生成 inventory-backend.jar

**3.3 启动后端服务**

**3.3.1 命令行启动**

方式一：使用Maven直接运行

> mvn spring-boot:run

方式二：运行打包后的JAR文件

> java -jar target/inventory-backend.jar

**3.3.2 验证服务状态**

启动成功后，控制台会显示类似以下信息：

> Started InventoryApplication in X.XXX seconds Application is running!
> Access URLs: Local: http://localhost:8080 Swagger:
> http://localhost:8080/swagger-ui.html

访问Swagger文档验证：

> 打开浏览器访问 http://localhost:8080/swagger-ui.html
>
> 如能正常显示API文档页面，说明后端启动成功

**第六章 常见问题排查**

**6.1 后端启动失败**

**问题：端口8080被占用**

现象：启动时报错 \"Port 8080 was already in use\"

解决方案：

> 方式一：修改后端端口
>
> 在 application.yml 中修改 server.port 为其他端口
>
> 方式二：关闭占用端口的程序
>
> netstat -ano \| findstr :8080 \# 查找占用进程 taskkill /PID 进程ID /F
> \# 结束进程

**问题：数据库连接失败**

现象：启动时报错 \"Communications link failure\"

解决方案：

> 1\. 确认MySQL服务已启动
>
> 2\. 检查数据库连接配置是否正确
>
> 3\. 确认数据库用户名密码正确
>
> 4\. 检查防火墙是否阻止了3306端口

**问题：Java版本不正确**

现象：启动时报错 \"Unsupported major.minor version\"

解决方案：

> 1\. 确认JDK版本为1.8
>
> 2\. 检查JAVA_HOME环境变量配置
>
> 3\. 运行 java -version 确认当前JDK版本

**6.2 前端启动失败**

**问题：npm install 失败**

解决方案：

> 1\. 删除 node_modules 目录和 package-lock.json
>
> 2\. 清除npm缓存：npm cache clean \--force
>
> 3\. 重新执行 npm install

**问题：Node Sass 编译失败**

解决方案：

> 1\. 安装构建工具：npm install -g windows-build-tools
>
> 2\. 或使用 dart-sass 替代 node-sass

**问题：端口8081被占用**

解决方案：

> 在 vue.config.js 中修改 devServer.port

**6.3 数据库连接失败**

常见原因及解决方案：

> 1\. MySQL服务未启动 → 启动MySQL服务
>
> 2\. 密码错误 → 检查配置文件中的密码
>
> 3\. 数据库不存在 → 按第二章创建数据库
>
> 4\. 防火墙阻止 → 添加MySQL到防火墙例外

**6.4 端口占用问题**

查看端口占用情况：

> netstat -ano \| findstr :端口号

结束占用进程：

> taskkill /PID 进程ID /F
